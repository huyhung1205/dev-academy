Bài 1

Giới thiệu lập trình và môi trường phát triển

Bài học mở đầu giúp bạn hiểu lập trình là gì, C++ được dùng ở đâu, vì sao cần compiler và IDE, và cách tự chuẩn bị môi trường để viết chương trình đầu tiên.

1. Giới thiệu

Khái niệm

Lập trình là việc viết chỉ dẫn để máy tính làm đúng các bước mà con người muốn.

Mục tiêu bài học

Biết C++ là gì, hiểu compiler và IDE, chạy được chương trình đầu tiên.

Kiến thức tiền đề

Chỉ cần biết mở máy tính, cài phần mềm cơ bản và sẵn sàng đọc từng bước.

2. Tài liệu lý thuyết

2.1 Lập trình là gì?

Khái niệm: Lập trình là quá trình biến ý tưởng thành các câu lệnh để máy tính thực hiện. Mỗi câu lệnh là một bước nhỏ, giống như hướng dẫn làm bánh hay chỉ đường đi học.

Tại sao cần học: Nếu không hiểu lập trình, bạn chỉ nhìn thấy code như một khối ký tự khó đọc. Khi hiểu bản chất, bạn sẽ biết chương trình được tạo ra từ các bước rõ ràng, chứ không phải “viết đại rồi chạy”.

Cách hoạt động: Con người nghĩ ra thuật toán, viết code theo cú pháp của C++, compiler dịch code, rồi hệ điều hành chạy chương trình trên máy.

Lỗi thường gặp: Người mới thường muốn nhảy ngay vào code mà không hiểu đề bài, dẫn đến viết sai logic. Một lỗi khác là quên đọc thông báo lỗi của compiler.

2.2 C++ và compiler

Khái niệm: C++ là ngôn ngữ lập trình mạnh, linh hoạt, phù hợp để học tư duy lập trình và cấu trúc dữ liệu.

Tại sao cần học: C++ giúp bạn luyện tư duy rõ ràng, quản lý dữ liệu tốt và hiểu cách máy tính xử lý chương trình ở mức thấp hơn.

Cú pháp cơ bản: 

#include <iostream>
using namespace std;

int main() {
    return 0;
}

Cách hoạt động: #include nạp thư viện, main() là điểm bắt đầu, return 0; báo chương trình kết thúc bình thường.

Lỗi thường gặp: Thiếu #include <iostream>, thiếu main(), gõ sai tên lệnh, hoặc quên dấu chấm phẩy.

2.3 IDE, terminal và quy trình làm việc

Khái niệm: IDE là công cụ hỗ trợ viết mã, biên dịch và chạy chương trình. Terminal là nơi bạn gõ lệnh để biên dịch nếu muốn làm thủ công.

Mục đích sử dụng: IDE giúp người mới đỡ bối rối, còn terminal giúp bạn hiểu rõ quy trình compile - run.

Cách làm việc cơ bản: Viết code → lưu file .cpp → biên dịch → chạy thử → xem lỗi → sửa → chạy lại.

Minh họa: Nếu bạn viết cout << "Xin chao";, chương trình chỉ in ra dòng chữ đó khi có main() và lệnh biên dịch thành công.

Lỗi thường gặp: Mở file nhưng chưa lưu, nhấn Run nhưng compiler báo thiếu file, hoặc chạy nhầm chương trình cũ.

2.4 Chương trình đầu tiên và thói quen viết code

Khái niệm: Chương trình đầu tiên thường in “Hello World” để kiểm tra môi trường hoạt động.

Tại sao cần học: Chạy được chương trình đầu tiên giúp bạn tin rằng môi trường đã cài đúng và hiểu được luồng compile - run.

Thói quen tốt: Đặt tên biến dễ hiểu, thụt lề đều, viết comment ngắn gọn và đọc lỗi theo từng dòng.

Ví dụ lỗi: Nếu thiếu return 0;, nhiều compiler vẫn chạy, nhưng khi học cơ bản bạn nên luôn viết đầy đủ để tạo thói quen tốt.

3. Ví dụ minh họa

Ví dụ cơ bản: Chương trình Hello World

#include <iostream>
using namespace std;

int main() {
    // In lời chào đầu tiên để kiểm tra môi trường C++
    cout << "Hello World" << endl;

    // Kết thúc chương trình bình thường
    return 0;
}

Chương trình làm gì: In ra màn hình dòng chữ Hello World.

Luồng thực thi: Compiler đọc file, tìm main(), thực hiện lệnh cout, rồi kết thúc với return 0;.

Kết quả đầu ra: Một dòng chữ Hello World xuất hiện trên màn hình console.

Ví dụ thực tế: Chào người dùng và in năm học

#include <iostream>
using namespace std;

int main() {
    // Khai báo tên người dùng để nhập từ bàn phím
    string name;
    int birthYear;

    // Yêu cầu người dùng nhập tên
    cout << "Nhap ten cua ban: ";
    getline(cin, name);

    // Yêu cầu người dùng nhập năm sinh
    cout << "Nhap nam sinh: ";
    cin >> birthYear;

    // In lời chào cá nhân hóa
    cout << "Xin chao " << name << "!" << endl;
    cout << "Ban dang hoc lap trinh C++ trong nam 2026." << endl;

    // Kết thúc chương trình
    return 0;
}

Chương trình làm gì: Nhận tên người dùng rồi chào bằng tên đó.

Luồng thực thi: Khai báo biến, nhập tên bằng getline, nhập năm sinh bằng cin, in lời chào, kết thúc.

Kết quả đầu ra: Console hiển thị lời chào có tên người dùng và một câu mô tả ngắn.

Câu 1. Vì sao sinh viên năm nhất nên học cách compiler và IDE làm việc trước khi học sâu C++?

Gợi ý: vì compiler và IDE là công cụ đầu tiên giúp bạn biến code thành chương trình chạy được. Hiểu chúng giúp bạn biết file nào cần lưu, lỗi nào do cú pháp, và cách sửa nhanh hơn thay vì đoán mò.

Câu 2. Nếu chương trình không chạy, bạn sẽ kiểm tra theo những bước nào?

Gợi ý: kiểm tra đã lưu file chưa, có main() chưa, có thiếu #include hoặc dấu ; không, đọc thông báo lỗi của compiler, rồi chạy lại sau mỗi lần sửa một lỗi.
