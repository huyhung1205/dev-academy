Bài 2

Thuật toán và mô tả thuật toán

Bài học này giúp bạn biết cách nghĩ trước khi code: xác định đầu vào, đầu ra, các bước xử lý, rồi mới chuyển logic sang C++.

1. Giới thiệu

Khái niệm

Thuật toán là chuỗi bước rõ ràng để giải một bài toán cụ thể.

Mục tiêu bài học

Biết mô tả thuật toán bằng lời, giả mã và chuyển thành chương trình C++.

Kiến thức tiền đề

Cần biết in màn hình, nhập dữ liệu cơ bản và đọc hiểu các câu lệnh đơn giản.

2. Tài liệu lý thuyết

2.1 Khái niệm thuật toán

Khái niệm: Thuật toán là tập hợp các bước có thứ tự để giải quyết một vấn đề.

Tại sao cần học: Nếu không có thuật toán, code dễ bị viết theo cảm tính và rất khó sửa khi sai.

Cách hoạt động: Bắt đầu từ dữ liệu đầu vào, xử lý từng bước, rồi tạo ra kết quả đầu ra.

Lỗi thường gặp: Bỏ sót bước, viết bước mơ hồ, hoặc nhầm lẫn giữa điều kiện và kết quả.

2.2 Mô tả bằng lời và giả mã

Khái niệm: Giả mã là cách viết thuật toán gần giống code nhưng chưa cần đúng cú pháp C++.

Tại sao cần học: Giúp bạn tập trung vào logic trước khi bị rối bởi dấu ngoặc, dấu chấm phẩy.

Cú pháp minh họa:

Nhập a, b
Nếu a > b thì in a
Ngược lại in b

Cách hoạt động: Người học đọc giả mã để hiểu thứ tự xử lý, sau đó chuyển từng bước thành lệnh `if`, `cin`, `cout`.

Lỗi thường gặp: Viết giả mã quá chung chung như “xử lý dữ liệu”, không nói rõ dữ liệu nào và xử lý ra sao.

2.3 Lưu đồ và kiểm thử thuật toán

Khái niệm: Lưu đồ là sơ đồ khối biểu diễn luồng xử lý của thuật toán.

Tại sao cần học: Lưu đồ giúp thấy rõ nhánh rẽ, vòng lặp và điểm kết thúc của bài toán.

Minh họa: Hình oval cho bắt đầu/kết thúc, hình thoi cho điều kiện, hình chữ nhật cho thao tác xử lý.

Cách kiểm thử: Hãy thử ít nhất 2 bộ dữ liệu: một bộ “đẹp” và một bộ “cận biên” để xem thuật toán có sai không.

Lỗi thường gặp: Không kiểm thử nên tưởng thuật toán đúng, nhưng đến khi gặp dữ liệu xấu thì kết quả sai.

2.4 Chuyển thuật toán thành C++

Khái niệm: Chuyển thuật toán thành chương trình nghĩa là biến từng bước logic thành câu lệnh C++ cụ thể.

Tại sao cần học: Đây là cầu nối giữa “nghĩ ra cách làm” và “viết được code chạy thật”.

Cách làm: Xác định biến cần dùng, nhập dữ liệu, dùng điều kiện hoặc vòng lặp, rồi in kết quả.

Lỗi thường gặp: Chuyển thiếu bước, hoặc dùng biến chưa được nhập giá trị.

3. Ví dụ minh họa

Ví dụ cơ bản: Tìm số lớn hơn giữa hai số

#include <iostream>
using namespace std;

int main() {
    // Khai báo hai số nguyên để nhập từ bàn phím
    int a, b;

    // Nhập số thứ nhất và số thứ hai
    cout << "Nhap so a: ";
    cin >> a;
    cout << "Nhap so b: ";
    cin >> b;

    // So sánh để tìm số lớn hơn
    if (a > b) {
        cout << "So lon hon la: " << a << endl;
    } else if (b > a) {
        cout << "So lon hon la: " << b << endl;
    } else {
        cout << "Hai so bang nhau." << endl;
    }

    // Kết thúc chương trình
    return 0;
}

Chương trình làm gì: So sánh hai số và in số lớn hơn hoặc thông báo hai số bằng nhau.

Luồng thực thi: Nhập `a`, nhập `b`, kiểm tra điều kiện `if`, rồi in kết quả tương ứng.

Kết quả đầu ra: Một thông báo cho biết số lớn hơn là số nào.

Ví dụ thực tế: Tính tổng 1 đến n

#include <iostream>
using namespace std;

int main() {
    // Khai báo biến n và tổng
    int n;
    int sum = 0;

    // Nhập giới hạn cần tính
    cout << "Nhap n: ";
    cin >> n;

    // Duyệt từ 1 đến n để cộng dồn
    for (int i = 1; i <= n; i++) {
        sum += i;
    }

    // In kết quả cuối cùng
    cout << "Tong tu 1 den " << n << " la: " << sum << endl;

    // Kết thúc chương trình
    return 0;
}

Chương trình làm gì: Tính tổng các số tự nhiên từ 1 đến n.

Luồng thực thi: Nhập `n`, khởi tạo `sum`, lặp từ 1 đến n, cộng dồn rồi in kết quả.

Kết quả đầu ra: Ví dụ nhập 5 thì kết quả là 15.

Câu 1. Tại sao cần viết thuật toán trước khi viết code?

Gợi ý: viết thuật toán trước giúp thấy rõ cách giải, phát hiện thiếu bước, tránh nhầm logic và giảm thời gian sửa lỗi khi chuyển sang C++.

Câu 2. Hãy mô tả bằng lời thuật toán tìm số lớn nhất trong 3 số.

Gợi ý: nhập 3 số, giả sử số đầu là lớn nhất, lần lượt so sánh với số còn lại để cập nhật giá trị lớn nhất, cuối cùng in ra kết quả.
