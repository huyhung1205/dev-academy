Bài 4

Toán tử và biểu thức

Bài học này giúp bạn hiểu cách C++ tính toán, so sánh và kết hợp nhiều điều kiện để tạo ra kết quả chính xác.

1. Giới thiệu

Khái niệm

Toán tử là ký hiệu thực hiện phép tính, còn biểu thức là sự kết hợp của toán tử và toán hạng.

Mục tiêu

Biết dùng toán tử số học, so sánh, logic và hiểu thứ tự ưu tiên khi tính toán.

Tiền đề

Nên nắm biến, kiểu dữ liệu và cách nhập xuất đơn giản từ bài trước.

2. Tài liệu lý thuyết

2.1 Toán tử số học

Khái niệm: Toán tử số học gồm `+`, `-`, `*`, `/`, `%`.

Tại sao cần học: Đây là nền tảng để tính tổng, hiệu, tích, thương và phần dư trong hầu hết bài toán.

Cách hoạt động: Compiler thực hiện phép tính theo thứ tự ưu tiên, rồi trả kết quả cho biểu thức.

Lỗi thường gặp: Chia hai số nguyên nhưng mong kết quả thập phân, hoặc quên rằng `%` chỉ dùng cho số nguyên.

2.2 Toán tử so sánh

Khái niệm: Toán tử so sánh kiểm tra hai giá trị có bằng nhau hay không, lớn hơn hay nhỏ hơn.

Tại sao cần học: Các câu lệnh điều kiện đều dựa vào kết quả đúng/sai của phép so sánh.

Cú pháp: `==`, `!=`, `>`, `<`, `>=`, `<=`.

Lỗi thường gặp: Nhầm `=` với `==`, dùng dấu so sánh sai thứ tự, hoặc so sánh số thực quá cứng nhắc.

2.3 Toán tử logic

Khái niệm: Toán tử logic kết hợp nhiều điều kiện bằng `&&`, `||`, `!`.

Tại sao cần học: Thực tế một quyết định thường không chỉ dựa trên một điều kiện duy nhất.

Cách hoạt động: `&&` chỉ đúng khi mọi điều kiện đúng, `||` đúng khi ít nhất một điều kiện đúng, `!` đảo giá trị logic.

Lỗi thường gặp: Viết điều kiện dài nhưng không dùng ngoặc nên đọc rất khó và dễ sai.

2.4 Ưu tiên và ép kiểu

Khái niệm: Biểu thức có thứ tự ưu tiên, ví dụ nhân chia thường trước cộng trừ.

Tại sao cần học: Nếu không hiểu thứ tự, bạn sẽ tính sai dù code không báo lỗi.

Minh họa: `2 + 3 * 4` cho ra 14, không phải 20.

Lỗi thường gặp: Quên dùng ngoặc khi muốn đổi thứ tự tính, hoặc quên ép kiểu khi cần số thực.

3. Ví dụ minh họa

Ví dụ cơ bản: Tính tiền mua hàng

#include <iostream>
using namespace std;

int main() {
    // Khai báo số lượng và đơn giá
    int quantity = 3;
    int price = 25000;

    // Tính thành tiền
    int total = quantity * price;

    // In kết quả ra màn hình
    cout << "Thanh tien: " << total << " VND" << endl;

    return 0;
}

Chương trình làm gì: Nhân số lượng với đơn giá để tính tổng tiền.

Luồng thực thi: Khai báo biến, thực hiện phép nhân, gán vào `total`, in kết quả.

Kết quả đầu ra: `75000 VND`.

Ví dụ thực tế: Kiểm tra sinh viên qua môn

#include <iostream>
using namespace std;

int main() {
    // Khai báo điểm chuyên cần và điểm thi
    double attendance, exam;

    // Nhập điểm
    cout << "Nhap diem chuyen can: ";
    cin >> attendance;
    cout << "Nhap diem thi: ";
    cin >> exam;

    // Tính điểm trung bình có trọng số
    double finalScore = attendance * 0.4 + exam * 0.6;

    // Kiểm tra điều kiện qua môn
    if (finalScore >= 5.0) {
        cout << "Sinh vien qua mon." << endl;
    } else {
        cout << "Sinh vien chua qua mon." << endl;
    }

    return 0;
}

Chương trình làm gì: Tính điểm trung bình có trọng số rồi kiểm tra qua môn.

Luồng thực thi: Nhập hai điểm, tính biểu thức, so sánh kết quả với ngưỡng 5.0.

Kết quả đầu ra: In trạng thái “qua môn” hoặc “chưa qua môn”.

6. Câu hỏi tự luận tự ôn tập

Câu 1. Vì sao cần phân biệt `=` và `==` trong C++?

Gợi ý: `=` là gán giá trị cho biến, còn `==` là so sánh. Nếu dùng sai, chương trình có thể chạy sai logic dù không báo lỗi cú pháp.

Câu 2. Hãy giải thích vì sao nên dùng ngoặc trong biểu thức phức tạp.

Gợi ý: ngoặc giúp làm rõ thứ tự tính toán, tránh phụ thuộc quá nhiều vào mức ưu tiên mặc định và giúp người đọc hiểu ngay ý đồ của bạn.
