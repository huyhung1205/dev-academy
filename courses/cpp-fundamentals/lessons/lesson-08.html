Hàm và cấu trúc chương trình

Hàm giúp chia chương trình thành các phần nhỏ, dễ đọc, dễ kiểm thử và dễ tái sử dụng.

1. Giới thiệu

Khái niệm

Hàm là một khối lệnh có tên, nhận tham số, thực hiện nhiệm vụ và có thể trả về kết quả.

Mục tiêu

Biết khai báo, định nghĩa, gọi hàm và hiểu sự khác nhau giữa tham trị và tham chiếu.

Tiền đề

Đã biết biến, kiểu dữ liệu, điều kiện và vòng lặp.

Video mô phỏng

Video mô phỏng sẽ được cập nhật sau.

2. Tài liệu lý thuyết

2.1 Khái niệm hàm

Khái niệm: Hàm là một phần chương trình làm một việc cụ thể như tính tổng, kiểm tra chẵn lẻ hay in thông báo.

Tại sao cần học: Khi chương trình lớn hơn, viết tất cả vào `main()` sẽ rất rối và khó sửa.

Cú pháp:

int add(int a, int b) {
    return a + b;
}

Cách hoạt động: Khi gọi hàm, chương trình nhảy vào phần định nghĩa, thực hiện lệnh bên trong rồi quay lại vị trí đã gọi.

Lỗi thường gặp: Quên `return`, quên kiểu trả về hoặc truyền sai tham số.

2.2 Tham số và giá trị trả về

Khái niệm: Tham số là dữ liệu đưa vào hàm, giá trị trả về là kết quả hàm gửi lại cho nơi gọi.

Tại sao cần học: Nhờ tham số và `return`, hàm có thể dùng cho nhiều trường hợp khác nhau.

Lỗi thường gặp: Không phân biệt kiểu dữ liệu của tham số, làm chương trình bị ép kiểu ngoài ý muốn.

2.3 Phạm vi biến

Khái niệm: Biến trong hàm chỉ dùng được trong phạm vi của nó.

Tại sao cần học: Hiểu phạm vi giúp tránh lỗi dùng nhầm biến ngoài tầm nhìn.

Lỗi thường gặp: Đặt tên biến giống nhau ở nhiều phạm vi làm người mới dễ nhầm.

2.4 Cấu trúc chương trình

Khái niệm: Chương trình tốt thường có `main()` điều phối và nhiều hàm con xử lý riêng.

Tại sao cần học: Cấu trúc rõ ràng giúp code dễ bảo trì và phù hợp khi làm dự án lớn hơn.

Lỗi thường gặp: Nhét mọi thứ vào `main()` hoặc gọi hàm trước khi khai báo mà không có prototype.

3. Ví dụ minh họa

Ví dụ cơ bản: Hàm tính tổng

#include <iostream>
using namespace std;

// Hàm cộng hai số nguyên
int add(int a, int b) {
    // Trả về tổng của hai tham số
    return a + b;
}

int main() {
    // Khai báo hai số cần tính
    int x = 5, y = 7;

    // Gọi hàm add để lấy kết quả
    int result = add(x, y);

    // In kết quả ra màn hình
    cout << "Tong la: " << result << endl;
    return 0;
}

Chương trình làm gì: Tính tổng hai số bằng hàm riêng.

Luồng thực thi: `main()` gọi `add()`, `add()` trả về tổng, `main()` in kết quả.

Kết quả đầu ra: `Tong la: 12`.

Ví dụ thực tế: Tính điểm trung bình

#include <iostream>
using namespace std;

// Hàm tính điểm trung bình của 3 môn
double averageScore(double a, double b, double c) {
    // Công thức trung bình cộng
    return (a + b + c) / 3.0;
}

int main() {
    // Khai báo điểm từng môn
    double math, physics, english;

    // Nhập điểm từ bàn phím
    cout << "Nhap diem Toan: ";
    cin >> math;
    cout << "Nhap diem Ly: ";
    cin >> physics;
    cout << "Nhap diem Anh: ";
    cin >> english;

    // Gọi hàm tính điểm trung bình
    double avg = averageScore(math, physics, english);

    // In kết quả
    cout << "Diem trung binh: " << avg << endl;
    return 0;
}

Chương trình làm gì: Tính điểm trung bình 3 môn học.

Luồng thực thi: Nhập dữ liệu, gọi hàm trung bình, in ra kết quả.

Kết quả đầu ra: Nếu nhập 8, 7, 9 thì kết quả là 8.

6. Câu hỏi tự luận tự ôn tập

Câu 1. Vì sao chương trình lớn nên tách thành nhiều hàm?

Gợi ý: tách hàm giúp chia nhiệm vụ rõ ràng, code gọn hơn, dễ kiểm thử từng phần và thuận tiện tái sử dụng khi cần.

Câu 2. Hãy giải thích sự khác nhau giữa tham số và giá trị trả về.

Gợi ý: tham số là dữ liệu đi vào hàm, còn giá trị trả về là kết quả mà hàm đưa ra cho nơi gọi. Hai phần này tạo nên sự trao đổi dữ liệu giữa các khối chương trình.
