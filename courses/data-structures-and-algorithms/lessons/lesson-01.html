Tổng quan DSA và độ phức tạp thuật toán

Bài mở đầu giúp người mới hiểu DSA là gì, vì sao phải học độ phức tạp và cách đọc Big O khi phân tích thuật toán.

Mục tiêu bài học

  • Độ phức tạp thời gian và không gian.
  • Big O: O(1), O(log n), O(n), O(n log n), O(n^2), O(2^n), O(n!).

  • Giới hạn thời gian trong bài chấm online.

Bài mở đầu giúp người mới hiểu DSA là gì, vì sao phải học độ phức tạp và cách đọc Big O khi phân tích thuật toán.

Trong bài này, điều quan trọng nhất là không chỉ nhớ định nghĩa mà còn phải biết bài toán nào nên dùng kỹ thuật nào, vì sao nó chạy nhanh hơn và khi nào không nên áp dụng. Khi đọc PDF gốc, hãy để ý các ví dụ và công thức vì chúng thường là điểm quyết định để làm bài tập.

Ở level của sinh viên năm nhất và người tự học, mục tiêu là hiểu thật rõ dữ liệu đang được lưu như thế nào, thao tác nào là O(1), thao tác nào là O(log n) hoặc O(n), rồi dùng kiến thức đó để chọn lời giải hợp lý hơn. Đây là tinh thần xuyên suốt toàn bộ khóa học.

Ví dụ minh họa

Ví dụ 1: Vòng lặp đơn

#include <iostream>
#include <algorithm>
using namespace std;

int main() {
    int n;
    cin >> n;
    long long sum = 0;
    for (int i = 1; i <= n; i++) {
        sum += i;
    }
    cout << sum << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Tính tổng 1..n để minh họa một vòng lặp đơn.
  • Input: Nhập n.
  • Output: In tổng 1..n.
  • Biến chính: Biến: n, sum, i.
  • Luồng chạy: Đọc n rồi cộng dồn.
  • Độ phức tạp / kết quả kỳ vọng: O(n), O(1).

Ví dụ 2: Hai vòng lặp lồng nhau

#include <iostream>
#include <algorithm>
using namespace std;

int main() {
    int n;
    cin >> n;
    int cnt = 0;
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= n; j++) {
            cnt++;
        }
    }
    cout << cnt << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Đếm số lần lặp của hai vòng lặp lồng nhau.
  • Input: Nhập n.
  • Output: In n * n.
  • Biến chính: Biến: n, cnt, i, j.
  • Luồng chạy: Vòng ngoài và vòng trong chạy n lần.
  • Độ phức tạp / kết quả kỳ vọng: O(n^2), O(1).

Câu hỏi tự luận tự ôn tập

Câu 1. Vì sao bài 1 quan trọng trong khóa học?

Gợi ý: bài học này là nền để hiểu các chủ đề nâng cao hơn, vì nó cho bạn cách nhìn đúng về dữ liệu, độ phức tạp và kỹ thuật giải bài.

Câu 2. Khi nào bạn sẽ chọn độ phức tạp trong một bài toán thực tế?

Gợi ý: hãy mô tả input, output, ràng buộc, rồi so sánh với đặc tính của độ phức tạp. Nếu nó giúp chương trình rõ ràng hơn, nhanh hơn hoặc tiết kiệm bộ nhớ hơn thì đó là lựa chọn hợp lý.
