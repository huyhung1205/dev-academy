Bài này ghép hai nền tảng quan trọng: hệ nhị phân, bát phân, thập lục phân và tư duy chia chương trình thành hàm nhỏ.

Trong bài này, điều quan trọng nhất là không chỉ nhớ định nghĩa mà còn phải biết bài toán nào nên dùng kỹ thuật nào, vì sao nó chạy nhanh hơn và khi nào không nên áp dụng. Khi đọc PDF gốc, hãy để ý các ví dụ và công thức vì chúng thường là điểm quyết định để làm bài tập.

Ở level của sinh viên năm nhất và người tự học, mục tiêu là hiểu thật rõ dữ liệu đang được lưu như thế nào, thao tác nào là O(1), thao tác nào là O(log n) hoặc O(n), rồi dùng kiến thức đó để chọn lời giải hợp lý hơn. Đây là tinh thần xuyên suốt toàn bộ khóa học.

Ví dụ minh họa

Ví dụ 1: Đổi hệ 10 sang hệ 2

#include <iostream>
#include <string>
#include <algorithm>
using namespace std;

int main() {
    int n;
    cin >> n;
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }
    string bin;
    while (n > 0) {
        bin.push_back(char('0' + (n % 2)));
        n /= 2;
    }
    reverse(bin.begin(), bin.end());
    cout << bin << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Đổi số thập phân sang nhị phân.
  • Input: Nhập n.
  • Output: In chuỗi nhị phân.
  • Biến chính: Biến: n, bin.
  • Luồng chạy: Chia liên tiếp cho 2 rồi đảo ngược.
  • Độ phức tạp / kết quả kỳ vọng: O(log n).

Ví dụ 2: Hàm tìm số lớn nhất và truyền tham chiếu

#include <iostream>
using namespace std;

int max3(int a, int b, int c) {
    int res = a;
    if (b > res) res = b;
    if (c > res) res = c;
    return res;
}

void swap2(int &a, int &b) {
    int temp = a;
    a = b;
    b = temp;
}

int main() {
    int x, y, z;
    cin >> x >> y >> z;
    cout << max3(x, y, z) << endl;
    swap2(x, y);
    cout << x << ' ' << y << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Minh họa prototype, hàm max3 và swap bằng tham chiếu.
  • Input: Nhập 3 số.
  • Output: In số lớn nhất rồi đổi chỗ x và y.
  • Biến chính: Biến: x, y, z, res, temp.
  • Luồng chạy: Gọi hàm riêng để xử lý rõ ràng.
  • Độ phức tạp / kết quả kỳ vọng: O(1).

Câu hỏi tự luận tự ôn tập

Câu 1. Vì sao bài 2 quan trọng trong khóa học?

Gợi ý: bài học này là nền để hiểu các chủ đề nâng cao hơn, vì nó cho bạn cách nhìn đúng về dữ liệu, độ phức tạp và kỹ thuật giải bài.

Câu 2. Khi nào bạn sẽ chọn hệ đếm và hàm trong một bài toán thực tế?

Gợi ý: hãy mô tả input, output, ràng buộc, rồi so sánh với đặc tính của hệ đếm và hàm. Nếu nó giúp chương trình rõ ràng hơn, nhanh hơn hoặc tiết kiệm bộ nhớ hơn thì đó là lựa chọn hợp lý.
