Mảng hai chiều, prefix sum và mảng hiệu

Mục tiêu bài học

  • Mảng hai chiều, hàng và cột.
  • Prefix sum 1D và 2D.
  • Mảng hiệu cho cập nhật đoạn.

Bài này mở rộng tư duy từ mảng 1D sang ma trận, rồi dùng prefix sum và mảng hiệu để tối ưu truy vấn và cập nhật đoạn.

Trong bài này, điều quan trọng nhất là không chỉ nhớ định nghĩa mà còn phải biết bài toán nào nên dùng kỹ thuật nào, vì sao nó chạy nhanh hơn và khi nào không nên áp dụng. Khi đọc PDF gốc, hãy để ý các ví dụ và công thức vì chúng thường là điểm quyết định để làm bài tập.

Ở level của sinh viên năm nhất và người tự học, mục tiêu là hiểu thật rõ dữ liệu đang được lưu như thế nào, thao tác nào là O(1), thao tác nào là O(log n) hoặc O(n), rồi dùng kiến thức đó để chọn lời giải hợp lý hơn. Đây là tinh thần xuyên suốt toàn bộ khóa học.

Ví dụ minh họa

Ví dụ 1: Nhập và in ma trận

#include <iostream>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    int a[100][100];
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) cin >> a[i][j];
    }
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) cout << a[i][j] << ' ';
        cout << endl;
    }
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Nhập và in lại ma trận 2 chiều.
  • Input: Nhập n, m và ma trận.
  • Output: In ma trận.
  • Biến chính: Biến: a, i, j.
  • Luồng chạy: Hai vòng lặp lồng nhau.
  • Độ phức tạp / kết quả kỳ vọng: O(nm).

Ví dụ 2: Prefix sum và mảng hiệu

#include <iostream>
using namespace std;

int main() {
    int n, q;
    cin >> n >> q;
    long long pre[1005] = {}, diff[1005] = {};
    for (int i = 1; i <= n; i++) {
        long long x;
        cin >> x;
        pre[i] = pre[i - 1] + x;
    }
    while (q--) {
        int l, r, val;
        cin >> l >> r >> val;
        diff[l] += val;
        diff[r + 1] -= val;
        cout << pre[r] - pre[l - 1] << endl;
    }
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Minh họa tổng đoạn và cập nhật đoạn.
  • Input: Nhập n, q và các truy vấn.
  • Output: In tổng đoạn rồi cập nhật đoạn.
  • Biến chính: Biến: pre, diff, l, r, val.
  • Luồng chạy: Prefix trả lời tổng, diff xử lý cập nhật.
  • Độ phức tạp / kết quả kỳ vọng: O(n + q).

Câu 1. Vì sao bài 5 quan trọng trong khóa học?

Gợi ý: hãy mô tả input, output, ràng buộc, rồi so sánh với đặc tính của mảng hai chiều và prefix. Nếu nó giúp chương trình rõ ràng hơn, nhanh hơn hoặc tiết kiệm bộ nhớ hơn thì đó là lựa chọn hợp lý.
