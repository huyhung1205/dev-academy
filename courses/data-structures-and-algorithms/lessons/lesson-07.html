Set và Map

Bài này ghép hai container rất mạnh của STL: set để loại trùng và map để lưu cặp key-value, đếm tần suất và gom nhóm dữ liệu.

Mục tiêu bài học

  • Set, multiset, unordered_set.
  • Map, multimap, unordered_map.
  • insert, find, count, erase, begin, rbegin.
  • Khi nào cần thứ tự và khi nào cần hash.

Bài này ghép hai container rất mạnh của STL: set để loại trùng và map để lưu cặp key-value, đếm tần suất và gom nhóm dữ liệu.

Trong bài này, điều quan trọng nhất là không chỉ nhớ định nghĩa mà còn phải biết bài toán nào nên dùng kỹ thuật nào, vì sao nó chạy nhanh hơn và khi nào không nên áp dụng. Khi đọc PDF gốc, hãy để ý các ví dụ và công thức vì chúng thường là điểm quyết định để làm bài tập.

Ở level của sinh viên năm nhất và người tự học, mục tiêu là hiểu thật rõ dữ liệu đang được lưu như thế nào, thao tác nào là O(1), thao tác nào là O(log n) hoặc O(n), rồi dùng kiến thức đó để chọn lời giải hợp lý hơn. Đây là tinh thần xuyên suốt toàn bộ khóa học.

Ví dụ minh họa

Ví dụ 1: Set loại trùng

#include <iostream>
#include <set>
using namespace std;

int main() {
    int n;
    cin >> n;
    set<int> se;
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        se.insert(x);
    }
    for (int x : se) cout << x << ' ';
    cout << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Loại bỏ phần tử trùng và sắp xếp tăng dần.
  • Input: Nhập n và n số.
  • Output: In các giá trị duy nhất.
  • Biến chính: Biến: se, x.
  • Luồng chạy: Chèn từng số vào set.
  • Độ phức tạp / kết quả kỳ vọng: O(n log n).

Ví dụ 2: Map đếm tần suất

#include <iostream>
#include <map>
using namespace std;

int main() {
    int n;
    cin >> n;
    map<int, int> mp;
    for (int i = 0; i < n; i++) {
        int x;
        cin >> x;
        mp[x]++;
    }
    for (auto it : mp) cout << it.first << ' ' << it.second << '\n';
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Đếm tần suất bằng map.
  • Input: Nhập n và dãy số.
  • Output: In cặp giá trị - tần suất.
  • Biến chính: Biến: mp, x.
  • Luồng chạy: Mỗi số tăng mp[x].
  • Độ phức tạp / kết quả kỳ vọng: O(n log n).
