Stack, queue và priority_queue

Bài này gộp stack, queue và priority_queue, giải thích LIFO, FIFO và cách lấy phần tử ưu tiên nhất trong C++ STL.

Mục tiêu bài học

  • Stack cho ngoặc đúng và biểu thức.
  • Queue cho hàng chờ và BFS.
  • Priority_queue cho max-heap và min-heap.

Bài này gộp stack, queue và priority_queue, giải thích LIFO, FIFO và cách lấy phần tử ưu tiên nhất trong C++ STL.

Trong bài này, điều quan trọng nhất là không chỉ nhớ định nghĩa mà còn phải biết bài toán nào nên dùng kỹ thuật nào, vì sao nó chạy nhanh hơn và khi nào không nên áp dụng. Khi đọc PDF gốc, hãy để ý các ví dụ và công thức vì chúng thường là điểm quyết định để làm bài tập.

Ở level của sinh viên năm nhất và người tự học, mục tiêu là hiểu thật rõ dữ liệu đang được lưu như thế nào, thao tác nào là O(1), thao tác nào là O(log n) hoặc O(n), rồi dùng kiến thức đó để chọn lời giải hợp lý hơn. Đây là tinh thần xuyên suốt toàn bộ khóa học.

Ví dụ minh họa

Ví dụ 1: Kiểm tra ngoặc đúng bằng stack

#include <iostream>
#include <stack>
#include <string>
using namespace std;

bool check(string s) {
    stack<char> st;
    for (char c : s) {
        if (c == '(') st.push(c);
        else if (c == ')') {
            if (st.empty()) return false;
            st.pop();
        }
    }
    return st.empty();
}

int main() {
    string s;
    cin >> s;
    cout << (check(s) ? "YES" : "NO") << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Kiểm tra chuỗi ngoặc tròn có cân bằng không.
  • Input: Nhập chuỗi ngoặc.
  • Output: In YES hoặc NO.
  • Biến chính: Biến: s, st.
  • Luồng chạy: Gặp '(' thì push, gặp ')' thì pop.
  • Độ phức tạp / kết quả kỳ vọng: O(n).

Ví dụ 2: Queue và priority_queue

#include <iostream>
#include <queue>
using namespace std;

int main() {
    queue<int> q;
    q.push(10);
    q.push(20);
    cout << q.front() << endl;
    q.pop();

    priority_queue<int> pq;
    pq.push(4);
    pq.push(9);
    cout << pq.top() << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Minh họa queue FIFO và priority_queue max-heap.
  • Input: Không cần input.
  • Output: In đầu queue và top của heap.
  • Biến chính: Biến: q, pq.
  • Luồng chạy: Push rồi lấy front/top.
  • Độ phức tạp / kết quả kỳ vọng: O(log n) với priority_queue.
