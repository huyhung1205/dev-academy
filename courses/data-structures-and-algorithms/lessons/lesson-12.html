Khái niệm và biểu diễn đồ thị

Bài này đặt nền cho đồ thị: định nghĩa, bậc đỉnh, đường đi, chu trình, liên thông và cách lưu đồ thị bằng ma trận kề hoặc danh sách kề.

Mục tiêu bài học

  • Đồ thị vô hướng, có hướng, đa đồ thị, giả đồ thị.
  • Đỉnh kề, cạnh liên thuộc, bậc và bán bậc.
  • Đường đi, chu trình, liên thông, SCC/weakly connected.
  • Ma trận kề, danh sách cạnh, danh sách kề.

Bài này đặt nền cho đồ thị: định nghĩa, bậc đỉnh, đường đi, chu trình, liên thông và cách lưu đồ thị bằng ma trận kề hoặc danh sách kề.

Trong bài này, điều quan trọng nhất là không chỉ nhớ định nghĩa mà còn phải biết bài toán nào nên dùng kỹ thuật nào, vì sao nó chạy nhanh hơn và khi nào không nên áp dụng. Khi đọc PDF gốc, hãy để ý các ví dụ và công thức vì chúng thường là điểm quyết định để làm bài tập.

Ở level của sinh viên năm nhất và người tự học, mục tiêu là hiểu thật rõ dữ liệu đang được lưu như thế nào, thao tác nào là O(1), thao tác nào là O(log n) hoặc O(n), rồi dùng kiến thức đó để chọn lời giải hợp lý hơn. Đây là tinh thần xuyên suốt toàn bộ khóa học.

Ví dụ minh họa

Ví dụ 1: Ma trận kề

#include <iostream>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    int a[105][105] = {};
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        a[u][v] = 1;
        a[v][u] = 1;
    }
    cout << a[1][2] << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Lưu và kiểm tra cạnh bằng ma trận kề.
  • Input: Nhập n, m và các cạnh.
  • Output: In 0 hoặc 1.
  • Biến chính: Biến: a, u, v.
  • Luồng chạy: Đánh dấu hai chiều trong đồ thị vô hướng.
  • Độ phức tạp / kết quả kỳ vọng: O(1) kiểm tra cạnh, O(n^2) bộ nhớ.

Ví dụ 2: Danh sách kề

#include <iostream>
#include <vector>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    vector<int> adj[1005];
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }
    cout << adj[1].size() << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Lưu đồ thị bằng danh sách kề.
  • Input: Nhập n, m và các cạnh.
  • Output: In số đỉnh kề với 1.
  • Biến chính: Biến: adj, u, v.
  • Luồng chạy: Mỗi đỉnh giữ một vector đỉnh kề.
  • Độ phức tạp / kết quả kỳ vọng: O(n + m).

Câu hỏi tự luận tự ôn tập

Câu 1. Vì sao bài 12 quan trọng trong khóa học?

Gợi ý: bài học này là nền để hiểu các chủ đề nâng cao hơn, vì nó cho bạn cách nhìn đúng về dữ liệu, độ phức tạp và kỹ thuật giải bài.

Câu 2. Khi nào bạn sẽ chọn đồ thị trong một bài toán thực tế?

Gợi ý: hãy mô tả input, output, ràng buộc, rồi so sánh với đặc tính của đồ thị. Nếu nó giúp chương trình rõ ràng hơn, nhanh hơn hoặc tiết kiệm bộ nhớ hơn thì đó là lựa chọn hợp lý.
