Bài này nối DSU với cây khung nhỏ nhất: biết Find/Union rồi dùng chúng trong Kruskal, sau đó so sánh với Prim.

Trong bài này, điều quan trọng nhất là không chỉ nhớ định nghĩa mà còn phải biết bài toán nào nên dùng kỹ thuật nào, vì sao nó chạy nhanh hơn và khi nào không nên áp dụng. Khi đọc PDF gốc, hãy để ý các ví dụ và công thức vì chúng thường là điểm quyết định để làm bài tập.

Ở level của sinh viên năm nhất và người tự học, mục tiêu là hiểu thật rõ dữ liệu đang được lưu như thế nào, thao tác nào là O(1), thao tác nào là O(log n) hoặc O(n), rồi dùng kiến thức đó để chọn lời giải hợp lý hơn. Đây là tinh thần xuyên suốt toàn bộ khóa học.

Ví dụ minh họa

Ví dụ 1: DSU cơ bản

#include <iostream>
using namespace std;

const int MAXN = 1005;
int parent[MAXN], sz[MAXN];

int Find(int u) {
    if (u == parent[u]) return u;
    return parent[u] = Find(parent[u]);
}

bool Union(int u, int v) {
    u = Find(u); v = Find(v);
    if (u == v) return false;
    if (sz[u] < sz[v]) swap(u, v);
    parent[v] = u;
    sz[u] += sz[v];
    return true;
}

int main() {
    for (int i = 1; i <= 4; i++) parent[i] = i, sz[i] = 1;
    Union(1, 2);
    Union(2, 3);
    cout << (Find(1) == Find(3)) << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Kiểm tra hai đỉnh có cùng tập hay không.
  • Input: Không cần input.
  • Output: In 1 nếu cùng tập.
  • Biến chính: Biến: parent, sz.
  • Luồng chạy: Find nén đường đi, Union gộp theo size.
  • Độ phức tạp / kết quả kỳ vọng: Gần O(1) trung bình.

Ví dụ 2: Kruskal và Prim

#include <iostream>
#include <vector>
#include <algorithm>
#include <queue>
using namespace std;

struct Edge { int u, v, w; };

int main() {
    vector<Edge> edges = {{1,2,1}, {1,3,4}, {2,3,2}, {2,4,3}};
    sort(edges.begin(), edges.end(), [](Edge a, Edge b) { return a.w < b.w; });
    int cost = 0;
    for (auto e : edges) cost += e.w;
    cout << cost << endl;
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<pair<int,int>>> pq;
    pq.push({0, 1});
    cout << pq.top().first << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Gợi ý cách nghĩ Kruskal và Prim.
  • Input: Không cần input.
  • Output: In tổng và phần tử đỉnh của heap.
  • Biến chính: Biến: edges, cost, pq.
  • Luồng chạy: Sort cạnh, rồi dùng heap lấy cạnh nhỏ nhất.
  • Độ phức tạp / kết quả kỳ vọng: O(E log E) và O(E log V).
