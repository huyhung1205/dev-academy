Trong bài này, điều quan trọng nhất là không chỉ nhớ định nghĩa mà còn phải biết bài toán nào nên dùng kỹ thuật nào, vì sao nó chạy nhanh hơn và khi nào không nên áp dụng. Khi đọc PDF gốc, hãy để ý các ví dụ và công thức vì chúng thường là điểm quyết định để làm bài tập.

Ở level của sinh viên năm nhất và người tự học, mục tiêu là hiểu thật rõ dữ liệu đang được lưu như thế nào, thao tác nào là O(1), thao tác nào là O(log n) hoặc O(n), rồi dùng kiến thức đó để chọn lời giải hợp lý hơn. Đây là tinh thần xuyên suốt toàn bộ khóa học.

Ví dụ minh họa

Ví dụ 1: Dijkstra

#include <iostream>
#include <algorithm>
#include <vector>
#include <queue>
using namespace std;

int main() {
    vector<pair<int,int>> adj[5];
    adj[1].push_back({2, 2});
    adj[1].push_back({3, 5});
    adj[2].push_back({3, 1});
    adj[2].push_back({4, 4});
    const int INF = 1e9;
    vector<int> dist(5, INF);
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<pair<int,int>>> pq;
    dist[1] = 0;
    pq.push({0, 1});
    while (!pq.empty()) {
        auto [du, u] = pq.top(); pq.pop();
        if (du != dist[u]) continue;
        for (auto [v, w] : adj[u]) if (dist[v] > dist[u] + w) {
            dist[v] = dist[u] + w;
            pq.push({dist[v], v});
        }
    }
    cout << dist[4] << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Tìm đường đi ngắn nhất bằng Dijkstra.
  • Input: Không cần input.
  • Output: In khoảng cách đến đỉnh 4.
  • Biến chính: Biến: adj, dist, pq.
  • Luồng chạy: Luôn relax cạnh từ đỉnh có dist nhỏ nhất hiện tại.
  • Độ phức tạp / kết quả kỳ vọng: O((V + E) log V).

Ví dụ 2: Bellman-Ford

#include <iostream>
#include <vector>
using namespace std;

struct Edge { int u, v, w; };

int main() {
    vector<Edge> edges = {{1,2,4}, {2,3,-2}, {1,3,5}};
    const int INF = 1e9;
    vector<int> dist(4, INF);
    dist[1] = 0;
    for (int i = 1; i <= 2; i++) {
        for (auto e : edges) if (dist[e.u] < INF && dist[e.v] > dist[e.u] + e.w) {
            dist[e.v] = dist[e.u] + e.w;
        }
    }
    cout << dist[3] << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Tìm đường đi ngắn nhất khi có cạnh âm.
  • Input: Không cần input.
  • Output: In khoảng cách đến đỉnh 3.
  • Biến chính: Biến: edges, dist.
  • Luồng chạy: Lặp n - 1 lần và relax toàn bộ cạnh.
  • Độ phức tạp / kết quả kỳ vọng: O(VE).

Ví dụ 3: Floyd-Warshall

#include <iostream>
#include <algorithm>
using namespace std;

const int INF = 1e9;
int d[4][4];

int main() {
    for (int i = 1; i <= 3; i++) for (int j = 1; j <= 3; j++) d[i][j] = (i == j ? 0 : INF);
    d[1][2] = 3;
    d[2][3] = 4;
    for (int k = 1; k <= 3; k++) {
        for (int i = 1; i <= 3; i++) {
            for (int j = 1; j <= 3; j++) {
                if (d[i][k] < INF && d[k][j] < INF) d[i][j] = min(d[i][j], d[i][k] + d[k][j]);
            }
        }
    }
    cout << d[1][3] << endl;
    return 0;
}

Giải thích chương trình

  • Chương trình làm gì: Tính khoảng cách giữa mọi cặp đỉnh.
  • Input: Không cần input.
  • Output: In khoảng cách từ 1 đến 3.
  • Biến chính: Biến: d, k, i, j.
  • Luồng chạy: Thử từng đỉnh trung gian.
  • Độ phức tạp / kết quả kỳ vọng: O(n^3).
