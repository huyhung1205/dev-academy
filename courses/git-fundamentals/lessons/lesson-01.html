Bài 1

Giới thiệu Git, cài đặt và repository

Bài đầu tiên giúp bạn nắm Git giải quyết vấn đề gì, cài đặt Git đúng trên Windows và hiểu repository là nền móng của mọi dự án.

Video mô phỏng

Video mô phỏng sẽ được cập nhật sau.

Mục tiêu học tập

  • Hiểu Git, version control và sự khác nhau giữa Git với GitHub
  • Cài đặt Git trên Windows và cấu hình tên, email
  • Biết tạo repository mới hoặc clone repository có sẵn

Tài liệu lý thuyết

Git và version control là gì

Git là hệ thống quản lý phiên bản phân tán. Version control giúp bạn lưu lại từng mốc thay đổi của dự án để có thể so sánh, xem lại hoặc quay về trạng thái cũ khi cần.

Git khác GitHub như thế nào

Git là công cụ chạy trên máy của bạn, còn GitHub là dịch vụ lưu trữ và cộng tác. Khi hiểu sự khác nhau này, bạn sẽ biết lúc nào dùng lệnh Git và lúc nào làm việc trên giao diện GitHub.

Cài đặt Git trên Windows

Sau khi cài Git, bạn thường dùng Git Bash hoặc Git Terminal để gõ lệnh. Bước quan trọng nhất là kiểm tra git --version để chắc chắn hệ thống đã nhận Git.

Repository là gì

Repository là nơi Git lưu mã nguồn và toàn bộ lịch sử thay đổi. Dự án mới thường bắt đầu bằng git init, còn dự án đã có sẵn trên GitHub thì thường bắt đầu bằng git clone.

Workflow thực tế

  • Cài Git và mở Git Bash.
  • Kiểm tra phiên bản, cấu hình user.name và user.email.
  • Tạo repository mới bằng git init hoặc tải repo có sẵn bằng git clone.

Lỗi thường gặp và cách khắc phục

  • Quên kiểm tra phiên bản sau khi cài Git.
  • Nhập sai email hoặc tên khi cấu hình commit.
  • Nhầm giữa repository local và remote.

Ví dụ minh họa

Ví dụ 1: Kiểm tra và cấu hình Git

Thiết lập Git đúng trước khi làm việc với dự án thật.

git --version
git config --global user.name "Nguyen Van A"
git config --global user.email "vana@example.com"

Output mẫu

git version 2.x.x

Giải thích lệnh

  • Mục đích: xác nhận Git đã cài xong và lưu tên, email cho commit.
  • Khi dùng: ngay sau khi cài Git hoặc khi đổi máy mới.
  • Kết quả mong đợi: Git in ra phiên bản và không báo lỗi khi cấu hình.
  • Lỗi thường gặp: sai email, quên --global, hoặc mở nhầm terminal không nhận lệnh.

Ví dụ 2: Tạo và tải repository

So sánh repo mới với repo có sẵn trên GitHub.

git init
git clone https://github.com/username/project.git
git remote -v

Output mẫu

Initialized empty Git repository... / Cloning into 'project'...

Giải thích lệnh

  • Mục đích: tạo repo mới hoặc tải repo có sẵn về máy.
  • Khi dùng: dự án mới thì git init, dự án có sẵn thì git clone.
  • Kết quả mong đợi: repo local được tạo và remote được gắn đúng.
  • Lỗi thường gặp: clone sai URL hoặc init ở thư mục sai.

Bài tập thực hành

Dễ

  • Nêu bằng lời Git dùng để làm gì trong dự án phần mềm.
  • Giải thích ngắn sự khác nhau giữa Git và GitHub.
  • Chạy git --version và ghi lại kết quả của bạn.

Trung bình

  • Mô tả local repository và remote repository bằng ví dụ.
  • Tự cài Git và cấu hình lại user.name, user.email.

Khó: Giải thích vì sao dự án mới nên tạo repo ngay từ đầu thay vì đợi đến cuối.

Câu hỏi tự luận tự ôn tập

1. Giải thích vì sao Git hữu ích với người mới bắt đầu học lập trình.

Git giúp người học lưu lại từng bước thay đổi, dễ quay lại khi sửa sai và luyện thói quen làm việc có tổ chức.

2. So sánh Git và GitHub bằng ví dụ dễ hiểu.

Git giống cuốn sổ lịch sử thay đổi nằm trên máy bạn, còn GitHub giống nơi gửi sổ đó lên để chia sẻ và cộng tác với người khác.

Tóm tắt

Sau bài này, bạn cần nhớ Git là công cụ quản lý phiên bản, GitHub là dịch vụ cộng tác và repository là nơi Git lưu lịch sử dự án.
