Branch, merge và xử lý conflict

Bài này giúp bạn làm việc song song an toàn, hợp nhất thay đổi và bình tĩnh xử lý xung đột khi nhiều người sửa cùng một chỗ.

Video mô phỏng

Video mô phỏng sẽ được cập nhật sau.

Mục tiêu học tập

  • Hiểu branch là gì và vì sao nên dùng branch
  • Tạo, chuyển và xóa branch đúng cách
  • Biết merge workflow và cách xử lý conflict

Tài liệu lý thuyết

Branch là gì

Branch là nhánh phát triển tách ra từ một commit. Nhờ branch, bạn có thể thử tính năng mới hoặc sửa lỗi mà không làm ảnh hưởng ngay đến nhánh chính.

Tạo, chuyển và xóa branch

Các lệnh như git branch, git switch và git switch -c giúp bạn tạo nhánh mới, chuyển giữa các nhánh và dọn dẹp branch đã hoàn thành.

Merge và merge conflict

Merge hợp nhất thay đổi từ branch này sang branch khác. Conflict xảy ra khi hai nhánh sửa cùng một đoạn và Git không tự quyết định được phần nào nên giữ.

Giải quyết conflict an toàn

Khi conflict xuất hiện, hãy mở file, đọc dấu đánh dấu, sửa thủ công, stage lại rồi commit. Đừng xóa bừa vì có thể làm mất thay đổi quan trọng.

Workflow thực tế

  • Tạo branch cho từng tính năng hoặc bug.
  • Commit trên branch đó cho tới khi xong việc.
  • Merge branch vào main, xử lý conflict nếu có và xóa branch đã hoàn thành.

Lỗi thường gặp và cách khắc phục

  • Làm việc lớn trực tiếp trên main.
  • Xóa branch trước khi chắc chắn đã merge xong.
  • Giải quyết conflict bằng cách xóa dấu đánh dấu nhưng quên kiểm tra nội dung.

Ví dụ minh họa

Ví dụ 1: Tạo branch tính năng

Tách một công việc riêng khỏi nhánh chính.

git switch -c feature/login-page
git branch

Output mẫu

* feature/login-page

Giải thích lệnh

  • Mục đích: tạo và chuyển sang branch mới để làm việc riêng.
  • Khi dùng: trước khi bắt đầu tính năng hoặc sửa lỗi lớn.
  • Kết quả mong đợi: branch hiện tại đổi sang tên mới.
  • Lỗi thường gặp: tạo branch nhưng quên chuyển sang branch đó.

Ví dụ 2: Merge và xử lý conflict

Hợp nhất branch và sửa xung đột nếu Git không tự làm được.

git switch main
git merge feature/login-page
git status

Output mẫu

CONFLICT (content): Merge conflict in index.html

Giải thích lệnh

  • Mục đích: đưa thay đổi từ branch tính năng về main.
  • Khi dùng: khi branch đã sẵn sàng và được review.
  • Kết quả mong đợi: merge thành công hoặc Git báo conflict để bạn xử lý.
  • Lỗi thường gặp: merge nhầm branch hoặc bỏ qua bước kiểm tra sau conflict.

Bài tập thực hành

Dễ

  • Tạo một branch mới và chuyển sang branch đó.
  • Liệt kê các branch hiện có.
  • Merge một branch nhỏ không xung đột vào main.

Trung bình

  • Mô tả các bước giải quyết một conflict trong file HTML.
  • Giải thích vì sao branch giúp dự án an toàn hơn.

Khó: Tạo một conflict nhỏ có chủ đích và giải quyết nó bằng cách giữ phần nội dung hợp lý nhất.

Trắc nghiệm

1. Branch dùng để làm gì?

2. Lệnh nào tạo branch mới và chuyển sang đó?

3. Merge dùng để làm gì?

4. Conflict xảy ra khi nào?

5. Sau khi merge xong, nên làm gì với branch cũ?

Câu hỏi tự luận tự ôn tập

1. Vì sao branch giúp dự án an toàn hơn?

Branch tách thay đổi ra khỏi nhánh chính nên nếu tính năng đang dở hoặc lỗi, main vẫn giữ trạng thái ổn định.

2. Hãy mô tả cách bạn xử lý conflict trong một file HTML.

Tôi mở file, đọc phần conflict, quyết định giữ phần nào, xóa dấu đánh dấu, stage lại và commit merge sau khi kiểm tra xong.

Tóm tắt

Branch và merge là nền tảng làm việc nhóm. Khi hiểu conflict, bạn sẽ xử lý hợp nhất thay đổi tự tin hơn rất nhiều.
