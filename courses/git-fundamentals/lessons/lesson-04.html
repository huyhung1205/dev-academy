GitHub cơ bản, Pull Request, .gitignore, SSH và GitHub Pages

Bài này nối Git với môi trường làm việc thật trên GitHub: remote, push, pull, Pull Request, .gitignore, SSH và GitHub Pages.

Video mô phỏng

Video mô phỏng sẽ được cập nhật sau.

Mục tiêu học tập

  • Kết nối local repository với remote trên GitHub
  • Dùng Pull Request, .gitignore, SSH và GitHub Pages
  • Biết quản lý file không nên commit trong dự án thật

Tài liệu lý thuyết

Remote, push và pull

Remote là bản sao repo ở nơi khác, thường là GitHub. git push đẩy commit lên remote, còn git pull kéo thay đổi mới về local để đồng bộ với nhóm.

Pull Request và review code

Pull Request là yêu cầu hợp nhất thay đổi từ branch này sang branch khác trên GitHub. Đây là nơi mô tả, review và thảo luận trước khi merge vào nhánh chính.

.gitignore và file không nên commit

.gitignore giúp Git bỏ qua những file tạm, file build, log hoặc dữ liệu nhạy cảm. Nếu lỡ commit nhầm, có thể dùng git rm --cached để ngừng theo dõi file đó.

SSH Key và GitHub Pages

SSH key giúp xác thực an toàn với GitHub mà không cần nhập mật khẩu mỗi lần. GitHub Pages thì dùng repository để triển khai website tĩnh và cập nhật bằng commit/push.

Workflow thực tế

  • Thêm remote, push branch lên GitHub và mở Pull Request.
  • Kiểm tra .gitignore để tránh commit file rác hoặc nhạy cảm.
  • Thiết lập SSH nếu muốn làm việc lâu dài và bật GitHub Pages cho website tĩnh.

Lỗi thường gặp và cách khắc phục

  • Push nhầm branch hoặc nhầm remote URL.
  • Commit file build hoặc file bí mật lên repo.
  • Đặt sai đường dẫn asset khi deploy GitHub Pages.

Ví dụ minh họa

Ví dụ 1: Push và mở Pull Request

Đưa thay đổi lên GitHub để người khác review.

git remote add origin https://github.com/username/project.git
git push -u origin feature/navbar

Output mẫu

branch 'feature/navbar' set up to track 'origin/feature/navbar'

Giải thích lệnh

  • Mục đích: đẩy branch lên GitHub và chuẩn bị mở PR.
  • Khi dùng: sau khi branch đã đủ nội dung để review.
  • Kết quả mong đợi: branch xuất hiện trên GitHub để tạo Pull Request.
  • Lỗi thường gặp: sai URL remote hoặc quên -u khiến branch không track.

Ví dụ 2: Bỏ qua file và deploy website

Quản lý file không nên commit và đưa site tĩnh lên GitHub Pages.

git rm --cached config.local.json
git add .
git commit -m "Ignore local config"
git push

Output mẫu

Removed from index, file vẫn còn trên máy

Giải thích lệnh

  • Mục đích: dừng theo dõi file nhạy cảm hoặc file sinh ra tự động.
  • Khi dùng: khi phát hiện repo có file không nên commit.
  • Kết quả mong đợi: file vẫn còn local nhưng không còn trong commit sau.
  • Lỗi thường gặp: nhầm giữa xóa khỏi repo và xóa khỏi máy.

Bài tập thực hành

Dễ

  • Mở repository trên GitHub và tìm nút Pull Request.
  • Tạo một file .gitignore cho dự án web tĩnh.
  • Thêm repository của bạn vào GitHub Pages và kiểm tra link công khai.

Trung bình

  • Giải thích vì sao .gitignore quan trọng trong dự án nhóm.
  • Mô tả sự khác nhau giữa git push và git pull.

Khó: Chuyển một repository từ HTTPS sang SSH và kiểm tra lại remote.

Câu hỏi tự luận tự ôn tập

1. Hãy mô tả quy trình mở Pull Request cho một tính năng nhỏ.

Tôi push branch lên GitHub, mở Pull Request, mô tả thay đổi, chờ review, sửa theo góp ý rồi merge khi được chấp nhận.

2. Vì sao SSH và .gitignore đều quan trọng trong dự án thật?

SSH giúp làm việc an toàn và tiện hơn, còn .gitignore giúp repo sạch, tránh file rác và hạn chế rủi ro lộ thông tin.
