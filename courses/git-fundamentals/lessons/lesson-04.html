Merge và xử lý xung đột

Merge hợp nhất thay đổi từ các nhánh lại với nhau. Nếu hai nhánh sửa cùng một chỗ, Git sẽ báo xung đột để bạn xử lý thủ công.

Video mô phỏng

Video mô phỏng sẽ được cập nhật sau.

Tài liệu lý thuyết

Khi một tính năng hoàn tất, bạn thường hợp nhất nhánh của mình về nhánh chính bằng lệnh merge. Nếu lịch sử phù hợp, Git có thể merge tự động.

git switch main
git merge login-page

Nếu cùng một dòng code bị sửa ở cả hai nhánh, Git sẽ chèn các dấu phân cách xung đột. Lúc đó bạn cần mở file, chọn nội dung đúng, rồi commit kết quả.

<<<<<<< HEAD
Nội dung trên main
=======
Nội dung từ branch
>>>>>>> login-page

Ví dụ minh họa

Nếu nhóm cùng sửa tiêu đề trang chủ, hãy thống nhất ai giữ phần nào trước khi merge để giảm xung đột.
