Git nâng cao, open source workflow và mini project

Bài cuối cùng gom các kỹ năng nâng cao để bạn tự tin sửa sai, tham gia open source và quản lý một dự án cá nhân hay dự án nhóm hoàn chỉnh.

Video mô phỏng

Video mô phỏng sẽ được cập nhật sau.

Mục tiêu học tập

  • Phân biệt git restore, git reset và git revert
  • Hiểu fork, clone, branch, push và Pull Request trong open source
  • Biết tổ chức mini project cá nhân hoặc nhóm bằng Git

Tài liệu lý thuyết

git restore, git reset, git revert

git restore thường dùng để khôi phục file hoặc bỏ stage, git reset là lệnh mạnh để quay lịch sử local và git revert tạo commit mới để đảo ngược một commit cũ an toàn hơn.

Open source workflow

Trong open source, bạn thường fork repository gốc, clone về máy, tạo branch riêng, commit, push lên fork rồi mở Pull Request về upstream để review.

Quản lý dự án cá nhân

Một dự án cá nhân tốt nên có commit nhỏ, branch rõ ràng, .gitignore đầy đủ và quy trình push thường xuyên lên GitHub để không mất dữ liệu.

Mini project thực tế

Mini project tốt nhất là một website tĩnh hoặc bài tập nhóm nhỏ để bạn thực hành đầy đủ từ local, branch, PR, merge đến deploy GitHub Pages.

Workflow thực tế

  • Xác định lỗi đang ở file chưa commit, commit local hay commit đã push.
  • Chọn lệnh hoàn tác phù hợp nhất với tình huống.
  • Fork/clone, branch, commit, push, PR và deploy cho mini project.

Lỗi thường gặp và cách khắc phục

  • Dùng reset --hard bừa bãi và làm mất dữ liệu.
  • Không sync fork với upstream trước khi làm open source.
  • Đợi đến cuối mới dùng Git cho dự án cá nhân.

Ví dụ minh họa

Ví dụ 1: Hoàn tác thay đổi đúng cách

Dùng lệnh phù hợp cho từng tình huống hoàn tác.

git restore index.html
git reset --soft HEAD~1
git revert a1b2c3d

Output mẫu

File được khôi phục / commit được đảo ngược an toàn

Giải thích lệnh

  • Mục đích: khôi phục file, quay lại commit local hoặc đảo ngược commit đã chia sẻ.
  • Khi dùng: tùy vào việc lỗi đang ở file, local commit hay commit public.
  • Kết quả mong đợi: sửa sai mà không làm rối lịch sử.
  • Lỗi thường gặp: dùng reset --hard khi chưa hiểu hậu quả.

Ví dụ 2: Open source workflow và mini project

Thực hành đóng góp và triển khai dự án hoàn chỉnh.

git clone https://github.com/yourname/project.git
git switch -c feature/readme
git add .
git commit -m "Improve README"
git push origin feature/readme

Output mẫu

Open Pull Request on GitHub

Giải thích lệnh

  • Mục đích: mô phỏng luồng fork/clone, branch, commit, push và PR.
  • Khi dùng: khi đóng góp open source hoặc làm mini project nhóm.
  • Kết quả mong đợi: có thay đổi rõ ràng, review được và dễ merge.
  • Lỗi thường gặp: push trực tiếp vào main hoặc mở PR quá lớn.

Bài tập thực hành

Dễ

  • Thử git restore trên một file nháp rồi kiểm tra lại trạng thái.
  • Mô tả quy trình fork, clone và mở PR.
  • Tạo kế hoạch Git cho mini project cá nhân của bạn.

Trung bình

  • Giải thích khác nhau giữa reset và revert.
  • Vẽ workflow làm việc nhóm có branch, PR và review.

Khó: Xây dựng một workflow Git hoàn chỉnh cho một website tĩnh và trình bày bằng sơ đồ.

Trắc nghiệm

1. Lệnh nào phù hợp để khôi phục một file?

2. Lệnh nào tạo commit mới để đảo ngược một commit cũ?

3. Open source workflow thường bắt đầu bằng gì?

4. Khi nào nên dùng git reset?

5. Mini project tốt nên có gì?

Câu hỏi tự luận tự ôn tập

1. Hãy so sánh git restore, git reset và git revert bằng lời của bạn.

git restore phù hợp khi chỉ cần khôi phục file, git reset mạnh hơn và tác động tới lịch sử local, còn git revert an toàn khi cần đảo ngược một commit đã chia sẻ.

2. Theo bạn, workflow open source chuẩn gồm những bước nào?

Workflow chuẩn gồm fork, clone, branch, commit, push, mở Pull Request, review và đồng bộ với upstream khi cần.
